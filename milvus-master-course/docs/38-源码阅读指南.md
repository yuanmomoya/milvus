# 38 源码阅读指南

## 学习目标

学完本章后，你应该能够：

- 了解 Milvus 源码的整体结构和语言选择。
- 定位核心模块的代码位置。
- 阅读写入链路和搜索链路的关键代码。
- 理解 Segment、Index 和 Compaction 的实现思路。
- 为贡献代码或深度排查问题打下基础。

## 项目概览

Milvus 是一个 Go + C++ 混合项目：

| 语言 | 职责 | 代码占比 |
|---|---|---|
| Go | 分布式调度、RPC、元数据管理、协调逻辑 | ~60% |
| C++ | 向量索引构建、搜索计算、SIMD 优化 | ~30% |
| Python | pymilvus SDK、测试 | ~10% |

```
milvus/
├── cmd/                    # 各组件入口（main.go）
├── internal/
│   ├── proxy/              # Proxy 组件
│   ├── rootcoord/          # RootCoord
│   ├── datacoord/          # DataCoord
│   ├── datanode/           # DataNode
│   ├── querycoord/         # QueryCoord
│   ├── querynodev2/        # QueryNode
│   ├── indexnode/          # IndexNode
│   └── storage/            # 存储抽象层
├── pkg/                    # 公共包
├── internal/core/          # C++ 核心（索引、搜索）
└── scripts/                # 构建脚本
```

## 核心模块定位

### 写入链路

```
用户 insert → Proxy (internal/proxy/impl.go)
  → 分配时间戳 (internal/tso/)
  → 写入 MQ (internal/mq/)
  → DataNode 消费 (internal/datanode/flow_graph/)
  → 形成 Segment (internal/datanode/segment/)
  → Flush 到 MinIO (internal/storage/)
```

关键文件：
- `internal/proxy/task_insert.go` — Insert 任务处理
- `internal/datanode/data_sync_service.go` — DataNode 数据同步
- `internal/datanode/segment_manager.go` — Segment 管理

### 搜索链路

```
用户 search → Proxy (internal/proxy/task_search.go)
  → 路由到 QueryNode (internal/querycoord/)
  → QueryNode 执行搜索 (internal/querynodev2/segments/)
  → C++ 索引搜索 (internal/core/src/query/)
  → 归并结果返回
```

关键文件：
- `internal/proxy/task_search.go` — Search 任务
- `internal/querynodev2/segments/segment.go` — Segment 搜索
- `internal/core/src/query/SearchOnIndex.cpp` — C++ 索引搜索

### 索引构建

```
DataCoord 触发 → IndexCoord 调度 → IndexNode 执行
  → 加载 Segment 数据
  → 调用 C++ 构建索引 (internal/core/src/index/)
  → 上传索引文件到 MinIO
```

关键文件：
- `internal/indexnode/task.go` — 索引构建任务
- `internal/core/src/index/VectorIndex.cpp` — 向量索引实现
- `internal/core/src/index/HNSWIndex.cpp` — HNSW 具体实现

## 阅读建议

### 入门路径

1. **从 Proxy 开始**：`internal/proxy/` 是所有请求的入口，代码相对简单
2. **跟踪一次 Insert**：从 `task_insert.go` 开始，跟踪数据流向
3. **跟踪一次 Search**：从 `task_search.go` 开始，理解查询分发和归并
4. **看 Segment 生命周期**：`internal/datacoord/segment_manager.go`

### 阅读技巧

```bash
# 克隆源码
git clone https://github.com/milvus-io/milvus.git
cd milvus

# 查找关键函数
grep -r "func.*Insert" internal/proxy/ --include="*.go"
grep -r "func.*Search" internal/querynodev2/ --include="*.go"

# 查看接口定义
find . -name "*.go" | xargs grep "type.*Interface" | head -20

# 查看 protobuf 定义（RPC 接口）
find . -name "*.proto" | head -10
```

## C++ 核心：索引实现

HNSW 索引的核心在 `internal/core/src/index/`，基于 hnswlib 库：

```cpp
// 简化的搜索逻辑
void HNSWIndex::Search(const float* query, int top_k, float* distances, int64_t* ids) {
    auto result = hnsw_->searchKnn(query, top_k, ef_);
    // 将结果写入 distances 和 ids
}
```

IVF 索引基于 Faiss 库：

```cpp
// 简化
void IVFFlatIndex::Search(const float* query, int top_k, int nprobe, ...) {
    faiss::SearchParameters params;
    params.nprobe = nprobe;
    index_->search(1, query, top_k, distances, ids, &params);
}
```

## 面试题

1. **Milvus 为什么用 Go + C++ 混合架构？** Go 适合分布式系统开发（goroutine、channel、RPC），C++ 适合高性能计算（SIMD、内存控制）。调度用 Go，计算用 C++。

2. **Proxy 在架构中的作用？** 接收所有客户端请求，做参数校验、鉴权、路由分发。它是无状态的，可以水平扩展。

3. **Segment 的状态转换是怎样的？** Growing（接收写入）→ Sealed（达到阈值）→ Flushed（持久化到 S3）→ Indexed（索引构建完成）。

## 小结

阅读 Milvus 源码的入口是 Proxy（Go），核心计算在 C++ 层。理解源码有助于深度排查问题和性能优化。建议从跟踪一次完整的 Insert 或 Search 请求开始，逐步深入各组件。
